# ⚾ 숫자 야구 게임 (Multiplayer Number Baseball)

## 📖 프로젝트 소개
언리얼 엔진 C++를 활용하여 구현한 멀티플레이어 기반의 숫자 야구 게임입니다. 서버와 클라이언트 간의 데이터 동기화를 통해 여러 플레이어가 동시에 참여할 수 있으며, 엄격한 입력 검사와 상태 관리 시스템을 통해 안정적인 게임 플레이를 제공합니다.

## ⚙️ 핵심 구현 기능

### 1. 게임 서버 및 판정 로직 (`GameModeBase`)
* **난수 생성:** 게임 시작 시 서버에서 1~9 사이의 중복되지 않는 3자리 난수를 정답으로 생성합니다. (예: `386`, `594`)
* **깐깐한 입력 검사 (유효성 검사):** * 입력값이 3자리 숫자인가?
    * 문자나 특수기호가 포함되지는 않았는가?
    * 중복되는 숫자가 존재하는가?
    * *💡 위 조건에 맞지 않을 경우, 플레이어의 기회를 소진하지 않고 안내 메시지를 출력합니다.*
* **판정 시스템:** 제시된 숫자와 정답을 비교하여 결과를 문자열(`1S1B`, `OUT` 등)로 반환합니다.
    | 판정 | 조건 |
    | :--- | :--- |
    | **S (Strike)** | 숫자와 자리가 모두 일치함 |
    | **B (Ball)** | 숫자는 존재하지만 자리가 일치하지 않음 |
    | **OUT** | 일치하는 숫자가 하나도 없음 |

### 2. 플레이어 상태 관리 (`PlayerState`)
* **기회 제한 시스템:** 각 플레이어에게 기본 3회의 타석(기회)을 부여합니다.
* **시도 횟수 동기화:** 채팅으로 숫자를 입력할 때마다 해당 플레이어의 시도 횟수를 1씩 증가시키고 관리합니다.
* **UI 피드백:** Getter 함수를 활용하여 플레이어가 숫자를 입력할 때마다 `[현재 시도 횟수 / 최대 시도 횟수]` 형식으로 현재 상태를 명확하게 표시합니다.

### 3. 게임 플로우 제어 (승리/무승부/리셋)
* **승리 판정:** 어떤 플레이어든 3번의 기회 안에 `3S`를 먼저 달성하면 즉시 해당 플레이어를 Winner로 판정하고, 승리 공지 위젯을 화면에 출력합니다.
* **무승부 판정:** 참가한 모든 플레이어가 3번의 기회를 모두 소모했음에도 정답자가 나오지 않으면 무승부로 판정하고 공지 위젯을 출력합니다.
* **게임 자동 리셋 시스템:** 승리 또는 무승부 결과가 공지된 후, 모든 플레이어의 데이터(시도 횟수 등)를 초기화하고 서버에서 새로운 정답 숫자를 생성하여 게임을 즉시 재시작합니다.

## 🛡️ 예외 처리 및 유효성 검사 테스트 결과

프로젝트의 안정성을 위해 다음과 같은 예외 상황을 테스트하고 처리했습니다.

* **잘못된 길이 입력:** `12` 입력 ➡️ `"3자리 숫자를 입력해주세요."` 출력 (기회 유지)
* **중복 숫자 입력:** `112` 입력 ➡️ `"중복되지 않은 숫자를 입력해주세요."` 출력 (기회 유지)
* **문자 포함 입력:** `A12` 입력 ➡️ 형식 오류 안내 및 기회 유지
* **정상 동작 흐름 (예시):**
    * 서버 정답: `386`
    * `Player A` 입력: `123` ➡️ `"0S1B"` 출력 (남은 기회: 2/3)
    * `Player B` 입력: `386` ➡️ `"3S0B"` 출력
    * 결과: `"플레이어 B 승리!"` 위젯 출력 후 게임 자동 리셋
